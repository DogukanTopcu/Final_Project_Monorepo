\section{Host Topology Summary}

The current host topology used by the platform can be summarized as follows:

\begin{itemize}
    \item one dedicated Google Cloud L4 host per SLM alias,
    \item one shared Google Cloud RTX PRO6000 host for the 20B--40B class models,
    \item one optional shared Nebius heavy host for the 70B and 120B class models, and
    \item a local control plane that communicates with all remote hosts through OpenAI-compatible HTTP endpoints.
\end{itemize}

\section{Benchmark Record Sketches}

The intended record shape for the planned coding benchmark is:

\begin{itemize}
    \item \texttt{id}
    \item \texttt{difficulty} = \texttt{easy} / \texttt{medium} / \texttt{hard}
    \item \texttt{prompt}
    \item \texttt{starter\_code}
    \item \texttt{public\_tests}
    \item \texttt{hidden\_tests}
    \item \texttt{topic}
    \item \texttt{source}
    \item \texttt{metadata}
\end{itemize}

The intended record shape for the human preference benchmark is:

\begin{itemize}
    \item \texttt{prompt\_id} and \texttt{prompt\_text}
    \item \texttt{architecture\_a}, \texttt{architecture\_b}
    \item \texttt{answer\_a}, \texttt{answer\_b}
    \item randomized display order
    \item selected winner or tie/skip
    \item evaluator/session identifier and timestamp
    \item latency, token, and cost metadata for both compared answers
\end{itemize}

\section{EATS Definition}

The project-specific EATS metric is defined as:

\[
\text{EATS} =
\frac{\text{accuracy}^2}
{\text{accuracy}^2 + \text{normalized\_cost}^{0.5}\,
\text{normalized\_algorithmic\_latency}^{0.3}\,\text{normalized\_energy}^{0.2}}
\]

Its purpose is to reward architectures that preserve useful answer quality while reducing normalized cost, algorithmic latency, and energy relative to a full-LLM baseline.
